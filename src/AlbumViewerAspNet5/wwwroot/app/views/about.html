About the Music Store Sample

Angular API Sample MVC Sample

This sample application demonstrates using a client centric JavaScript framework to communicate with a Web Connection backend via JSON. This is a common usage pattern today that uses the Web Browser as a rich client to provide an interactive user interface connected to data served from a Web Server. This example uses AngularJs, Bootstrap css and the web connection wwRestProcess class to provide this client centric and mobile aware, Web based sample.

Mobile

The sample is built using Bootstrap using mobile-first design mode. What this means is that the site is adaptive and works well on mobile devices as well as on large screens. You should be able to access this sample with just about any reasonably recent smartphone browser and get a decent usability experience. If you want to access the site with a mobile phone you can use the QR code on the right to scan and launch the site on your phone.

Two Versions: Service and MVC

There are two versions of this sample: An Angular application that uses an API back-end and a second incomplete version that uses MVC. Both controllers look very similar with the difference of returing either raw data results or API results.

AngularJs Front End

The client application is a pure JavaScript based front end launched from a plain .html page (no server code in the HTML). The front end uses AngularJs UI presentation, data-binding and accessing of the remote HTTP service to pull down the application data. AngularJs's strongly opinionated framework and especially the databinding and modularization of client side JavaScript code make it easy to create rich front ends to backend data as demonstrated by this small sample application.

Note that this application purposefully mixes a few different approaches in some of the modules to demonstrate different ways that you can do things. The albums views use the albumsService to explicitly isolate 'data' specific service operations and 'business logic'. The Artist views opt for the simpler direct access to data from controllers, which - while considered bad practice for testing and reusability - does make for a lot less code and easier to follow self-contained logic. The latter approach is valid for isolated or simple components/pieces in applications.
